### 2026-05-10
- review-false-positive: `.colSums(M, n, n)` and `.rowSums(M, m, n)` are documented base R primitives (less-checking variants of `colSums`/`rowSums`). They appear "internal" but live in `baseenv()` and are stable. A static review flagged them as undefined in `R/centralities.R`; running `centralities_htna()` end-to-end on `human_long`/`ai_long` produces non-NA values for both `Clustering` and `BetweennessRSP`, confirming the helpers (`.weighted_clustering`, `.rsp_betweenness`) work as written.
- legend-overlay: the actor-overlay legend system in `R/sequence_plot_htna.R` only renders for `position %in% c("right", "bottom")` because `.htna_legend_gutter_fig()` only computes gutter geometry for those two. Other base-graphics legend specifiers (`"left"`, `"topleft"`, `"top"`, `"center"`, …) used to crash via `match.arg`. Now normalized through `.htna_normalize_overlay_pos()` which collapses to `"bottom"` for top/bottom-anchored specs and `"right"` otherwise (with warning).
- palette-recycling: `htna_palette` has 6 colours. Until this session, `palette[seq_along(actors)]` produced silent `NA` fills when actor count exceeded 6 — broken legends and missing geom layers were the result. The new internal `.htna_actor_colors(n_or_levels)` recycles with one warning. Used at all 5 sites that previously indexed the palette directly.
- gap-semantics: `.discover_paths()` previously rewrote `gap_vals == 0L` to `1L` silently when `type = "gapped"`. This conflated "contiguous" and "gapped" semantics. Now errors with a message pointing the user at `type = "contiguous"`.
- ggplot2-guards: `ggplot2` is in `Suggests:` (not `Imports:`). Every function that uses `ggplot2::` symbols needs `requireNamespace("ggplot2", quietly = TRUE)` before the first ggplot call. Currently guarded: `plot_centralities()`, `plot_frequencies_htna()` (added this session). `sequence_plot_htna()` does *not* need a guard — it uses base graphics (`graphics::par`, `graphics::legend`), not ggplot2.
- nestimate-upstream-bug: `Nestimate 0.5.4` (also the version on `mohsaqr.r-universe.dev`) declares S3 methods `print.wtna_perm_mixed` and `summary.wtna_perm_mixed` in its NAMESPACE without defining the implementations. The warning surfaces when htna installs/loads and is echoed across `R CMD check` in 3 WARNINGs and 4 NOTEs (every check that scans NAMESPACE / S3 dispatch / Rd usage will repeat it). The same release is missing the internal `.param_get()` that `Nestimate:::.reliability_association()` calls — this manifested as 3 test failures hitting `htna::reliability_htna()`. Documented for CRAN reviewers in `cran-comments.md`. Worked around in tests via `skip_if_nestimate_param_get_missing()` in `tests/testthat/helper-fixtures.R`. The `.param_get` undefined-global NOTE is silenced via `utils::globalVariables(".param_get")` in `R/htna-package.R`.
- alias-rd-docs: A pure alias defined as `<name> <- pkg::fn` produces an Rd file whose `\usage` block lists every formal parameter of the upstream function but no `@param` blocks of its own — CRAN's "Undocumented arguments" warning fires. The fix is `@inheritParams pkg::fn` in the alias's roxygen header; that pulls the upstream documentation into the local Rd. Applied to all 8 `_htna` aliases this session. Same Rd files needed `@examples` for CRAN — added with `\donttest{}` for runs that take > a few seconds.
- inst-wordlist: `spelling::spell_check_package(".")` reads `inst/WORDLIST` for legitimate technical terms and locale variants. Created one to whitelist British English prose (colour/behaviour/visualise/standardise/recognise/optimise/...) plus package-specific terms (Nestimate, cograph, htna, RSP, Markovian, treemap, splot, ...). Keep `Language: en-US` in DESCRIPTION since R parameter / variable names use US spelling (`color`, etc.) — the WORDLIST handles the prose layer.
- url-trailing-slash: `urlchecker::url_check()` flags `https://sonsoles.me/htna` as redirecting to `https://sonsoles.me/htna/` (trailing slash). CRAN rejects redirects, so the canonical form (with trailing slash) must be in DESCRIPTION's `URL:` field. roxygen propagates this to `man/htna-package.Rd` on next `document()`.
