### Deviations from the agent note

The note specs `node_size = 7`, `edge_color`, `edge_label_size`,
`edge_label_position` in `.htna_style_defaults()`. cograph 2.1.2’s
actual chain is `plot_htna → tplot → splot`, and `tplot` translates
*its* dotted formal names (`vsize`, `edge.color`, `edge.label.cex`,
`edge.label.position`) into splot’s underscored args before
concatenating `...`. Passing the underscored names from htna therefore
**collides** with what tplot has already set —
`formal argument "X" matched by multiple actual arguments`.

The fix in `R/style_defaults.R` is to use tplot’s formal names so
they’re absorbed cleanly:

| Note says | Working impl uses | Why |
|----|----|----|
| `node_size = 7` | `vsize = 7` | tplot maps `vsize → splot_args$node_size` |
| `edge_color` | `edge.color` | tplot maps conditionally |
| `edge_label_size` | `edge.label.cex` | tplot always sets |
| `edge_label_position` | `edge.label.position` | tplot always sets |

When cograph 2.2 lands and `plot_htna` calls `splot` directly (skipping
tplot), revert these to the underscored names the note specs.

## Blocker

`R/reexports.R` exists in the working tree (untracked, dated 2026-04-27)
with a **competing** `plot_htna` definition. It bypasses
[`cograph::plot_htna`](https://sonsoles.me/cograph/reference/plot_htna.html)
entirely and calls
[`cograph::splot()`](https://sonsoles.me/cograph/reference/splot.html)
directly with a custom layout matrix, a custom palette (`htna_palette`:
`#FFD966 #B4A7D6 #93C47D #E06666 #6FA8DC #F6B26B`), darkened donut
borders (`.darken()`), and a custom
[`graphics::legend()`](https://rdrr.io/r/graphics/legend.html) renderer.
R loads files alphabetically, so `reexports.R` overwrites the Section 4
wrapper from `R/plot_htna.R`.

This file is **not in HEAD** — HEAD’s `reexports.R` is the original
3-line
[`cograph::plot_htna`](https://sonsoles.me/cograph/reference/plot_htna.html)
re-export. Someone (likely Sonsoles, given timestamps) wrote an
alternative implementation between the agent-note draft and now.

**Decision needed before this can ship:**

1.  **Adopt the agent note as written.** Delete `reexports.R`; keep
    `R/plot_htna.R`. Loses: custom palette, donut darkening, custom
    legend, left-starting circular layout. Wins: the rest of the
    architecture (`.htna_style_defaults`, layout port, bootstrap/diff
    wrappers) all compose with
    [`cograph::plot_htna`](https://sonsoles.me/cograph/reference/plot_htna.html)
    as the entry point — which the note spent Section 1 arguing is the
    right choice.

2.  **Adopt `reexports.R` as written.** Delete `R/plot_htna.R`. The
    note’s architecture (Sections 3–7) becomes inert — the bootstrap and
    diff wrappers in particular call
    [`cograph::plot_htna`](https://sonsoles.me/cograph/reference/plot_htna.html)
    and would need rewriting against the splot-direct path.

3.  **Merge.** Keep the Section 4 wrapper, fold the visual choices from
    `reexports.R` into `.htna_style_defaults()`:

    - palette → `defaults$group_colors`
    - donut darkening → add `defaults$donut_color`
    - left-starting circular → flip the layout port to start at `pi`
      instead of `pi/2`
    - custom legend → drop (cograph’s `legend = TRUE` +
      `legend_position = "bottom"` is the note’s choice)

    This is the option I’d recommend if no one has strong feelings about
    bypassing `plot_htna`. It preserves the note’s anti-coupling
    argument (Section 1) while keeping the visual identity Sonsoles
    already chose.

## Quirks discovered along the way

Worth recording for future-you and adding to the agent note’s Section 11
quirks list:

1.  **`importFrom(cograph, plot_htna)` shadows our local `plot_htna`**
    in the htna namespace. The wrapper in `R/plot_htna.R` calls
    [`cograph::plot_htna()`](https://sonsoles.me/cograph/reference/plot_htna.html)
    namespaced, so the import is unnecessary and harmful. Already
    removed.

2.  **tplot’s name translation table** (Section “Deviations” above) is
    the single biggest gotcha when writing wrappers. Any default that
    uses splot’s underscored name will collide if tplot also sets it.
    The safe-by-construction approach is: pass tplot’s preferred name
    when tplot has a formal for the concept, and use splot’s name only
    for things tplot doesn’t touch (`arrow_size`, `edge_start_length`,
    `edge_start_style`, `legend_horiz`, `edge_label_style`,
    `edge_label_leading_zero`).

3.  **`splot.netobject` sets `tna_styling = TRUE` by default** when the
    network’s method is in
    `c("relative", "frequency", "attention", "co_occurrence", "wtna", "wtna_cooccurrence")`.
    This activates a whole block of internal defaults inside splot
    (lines 196–235 of
    [`cograph::splot`](https://sonsoles.me/cograph/reference/splot.html)).
    Our defaults intentionally re-set most of these so the htna look is
    deterministic regardless of `method`, but a future change to splot’s
    `.tna_style_defaults` could silently change htna visuals.
